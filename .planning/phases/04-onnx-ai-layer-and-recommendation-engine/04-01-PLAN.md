       - Constructor takes db_pool (asyncpg.Pool)

         ```sql
         SELECT sensor_type, AVG(value) AS mean_val, MIN(value) AS min_val,
                MAX(value) AS max_val, STDDEV(value) AS std_val, COUNT(*) AS reading_count
         FROM sensor_readings
         WHERE zone_id = $1 AND quality = 'GOOD' AND time > NOW() - ($2 || ' hours')::interval
         GROUP BY sensor_type
         ```

       - async def aggregate_zone_features(zone_id: str, sensor_types: list[str], window_hours: int = 24) -> dict[str, dict[str, float]] | None:
         Runs FEATURE_QUERY, returns {sensor_type: {mean_val, min_val, max_val, std_val, reading_count}} or None if total GOOD readings < MIN_READINGS.
       - def build_feature_vector(aggregated: dict[str, dict[str, float]], sensor_types: list[str]) -> list[float]:
         Converts aggregated dict into a flat float list. For each sensor_type in sensor_types, appends [mean, min, max, std]. If a sensor_type is missing from aggregated, appends [float('nan')] * 4. This guarantees a fixed-length vector (len = len(sensor_types) * 4).
       - async def check_data_maturity(zone_id: str) -> dict:
         Runs MATURITY_GATE_QUERY from 04-RESEARCH.md:
         ```sql
         SELECT COUNT(*) FILTER (WHERE quality = 'GOOD')::float / NULLIF(COUNT(*), 0) AS good_ratio,
                MIN(time) AS earliest_reading, NOW() - MIN(time) AS data_span
         FROM sensor_readings WHERE zone_id = $1 AND time > NOW() - INTERVAL '4 weeks'
         ```
         Returns {"zone_id": zone_id, "good_ratio": float, "data_span_days": float, "gate_passed": bool}
         gate_passed = good_ratio >= 0.8 AND data_span_days >= 28

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | generate_synthetic_data.py | mitigate | Mark synthetic rows — all synthetic data uses quality flags including SUSPECT/BAD, but is inserted into the same sensor_readings table. Synthetic data is a dev tool only. Production hub has real data. |
| T-04-02 | Denial of Service | feature_aggregator.py | mitigate | MIN_READINGS threshold prevents inference on insufficient data. check_data_maturity prevents activation on sparse zones. |
| T-04-03 | Information Disclosure | feature_aggregator.py | accept | Feature vectors contain sensor aggregates — low sensitivity data on a LAN-only system. No PII. |
