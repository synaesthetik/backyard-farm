## Color

All values from `hub/dashboard/src/app.css`. No new color values introduced in Phase 4.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` (`--color-bg`) | Page background |
| Secondary (30%) | `#1c1f2b` (`--color-surface`) / `#242838` (`--color-surface-hover`) | AI Status card surface, toggle control background, settings page cards |
| Accent (10%) | `#4ade80` (`--color-accent`) | Reserved list below |
| Destructive | `#ef4444` (`--color-offline`) | Maturity gate failure indicator; "below threshold" warning state |

Accent (`#4ade80`) reserved for:
1. Active/approved state of AI toggle (when ONNX is enabled for a domain)
2. Maturity progress bar fill when a domain has reached the 4-week gate
3. Recommendation count when approval rate is >= 90%
4. CommandButton approve variant (carried forward — no change)
5. Active tab indicator in TabBar (carried forward — no change)
6. WebSocket connected dot in header (carried forward — no change)

Semantic color assignments for AI Status card states:
- `--color-accent` (#4ade80): domain mature, ONNX active
- `--color-stale` (#f59e0b): domain in progress (data accumulating, < 4 weeks)
- `--color-offline` (#ef4444): domain blocked (below 80% GOOD-flag ratio, gate failed)
- `--color-muted` (#6b7280): domain not yet started or rules-mode active

Accent-dim (`rgba(74, 222, 128, 0.12)` / `--color-accent-dim`): progress bar track background when domain is mature.

## Component Inventory

### New Components (Phase 4)

#### 1. AIStatusCard.svelte
Location: `hub/dashboard/src/lib/AIStatusCard.svelte`
Placement: Home tab (`+page.svelte`), inserted between FlockSummaryCard and zone grid. On desktop (>= 640px), occupies the flock column below FlockSummaryCard.

Structure:
```
AIStatusCard
  card header: "AI Engine" (20px heading) + overall status badge
  three domain rows (irrigation, zone health, flock anomaly):
    DomainMaturityRow (repeating sub-component)
      domain label (14px Inter)
      progress bar (8px height, track + fill)
      progress text (14px, e.g. "3 wk / 4 wk" or "Mature")
      mode badge (AI | RULES — 11px pill, HealthBadge pattern)
      approval rate (14px, shown only when recommendation_count > 0)
  cold-start message (italic, 14px, --color-muted): shown when any domain is immature
```

States:
- **Cold start (all domains immature):** Card is prominent. Italic message: "The AI is still learning — rule-based recommendations are active." Progress bars show week-by-week fill. Mode badges show "RULES".
- **Partial maturity (some domains mature):** Mature domains show green fill + "AI" badge. Immature show amber fill + "RULES" badge.
- **Fully mature (all domains active):** Card collapses to a compact summary row (heading + all-green badge). No progress bars rendered. This minimized state prevents visual clutter once models are running.
- **Gate failure (zone below 80% GOOD-flag ratio):** Domain row shows red fill, "BLOCKED" badge (11px pill, `--color-offline`), text: "Below quality threshold — check sensor calibration."
- **Loading:** Skeleton shimmer using `--color-border-subtle` background animation on progress bar track.

Card border: `1px solid var(--color-border)`. No left-accent border (reserved for RecommendationCard).

#### 2. AISettingsToggle.svelte
Location: `hub/dashboard/src/lib/AISettingsToggle.svelte`
Placement: New settings route `/settings/ai` (or `/settings` with AI section if settings page is consolidated).

Structure:
```
AISettingsToggle
  label (16px): domain name (e.g., "Irrigation Recommendations")
  description (14px, --color-muted): one sentence explaining what ONNX controls for this domain
  toggle switch: "AI" (accent, left) / "Rules" (muted, right) — visually distinct states
  disabled state: toggle disabled + tooltip "Model not yet mature" when domain has not passed gate
```

Three instances rendered: irrigation, zone health, flock anomaly. Each is independently controllable.

Toggle behavior:
- Active (ONNX on): toggle track filled with `--color-accent`, thumb right
- Inactive (Rules on): toggle track `--color-border`, thumb left
- Disabled (not mature): opacity 0.5, cursor not-allowed, no interaction
- ARIA: `role="switch"` + `aria-checked` + `aria-disabled`

Change takes effect on next inference cycle — no spinner needed. Toast: "AI enabled for irrigation" or "Rules restored for zone health" on toggle.

#### 3. DomainMaturityRow.svelte (sub-component)
Internal to AIStatusCard. Not exported for standalone use.

Progress bar specs:
- Track: 100% wide, 8px height, border-radius 4px, background `--color-border`
- Fill: transition width 0.6s ease, border-radius 4px
- Fill colors: amber (`--color-stale`) when 0-79% of 4-week target; green (`--color-accent`) when 80-100%; red (`--color-offline`) when gate failed

### Existing Components Modified (Phase 4)

#### RecommendationCard.svelte — source badge
Add an engine source badge to the bottom-right of each recommendation card: "AI" or "RULES" (11px pill, same pattern as HealthBadge). This allows the farmer to see which engine generated a given recommendation.

- AI badge: `--color-accent` background, `--color-bg` text
- RULES badge: `--color-border` background, `--color-text-secondary` text

No layout changes. Badge added inside `.controls` div, left-aligned before the CommandButton pair.

#### Home tab (+page.svelte) — AIStatusCard insertion
Insert AIStatusCard into the flock column on the Home tab. On mobile (single column): AIStatusCard renders above FlockSummaryCard. On desktop (2-column): AIStatusCard stacks below FlockSummaryCard in the right column. Once all domains are mature, AIStatusCard collapses to compact summary (no layout shift — compact mode still occupies the slot).

#### App header — settings icon button
A settings icon button (lucide-svelte `Settings` icon) is added to the app header to provide navigation to `/settings/ai`. The button must include `aria-label="Open AI settings"` on the icon button element. Touch target minimum 44px height, consistent with existing header interactive elements.

## Interaction Contracts

### Recommend-and-confirm UX (unchanged)
ONNX recommendations appear in the existing `/recommendations` queue with identical RecommendationCard UX. The approve/reject flow, deduplication, and back-off window are unchanged from Phase 2. The only addition is the "AI" source badge on each card. Farmers do not need to learn a new flow.

### Toggle (AISettingsToggle) interaction sequence
1. Farmer taps toggle in `/settings/ai`
2. Optimistic UI update: toggle flips immediately
3. PATCH `/api/settings/ai` with `{ domain: "irrigation", mode: "ai" | "rules" }`
4. On success: toast "AI enabled for irrigation" (2.5s duration)
5. On failure: toggle reverts, toast "Settings update failed — check connection"
6. No page reload, no restart required

### Data maturity progress refresh
AIStatusCard data sourced from WebSocket snapshot extension (`ws.svelte.ts`). Server pushes maturity state on WebSocket connection and after each inference cycle (15-minute max interval for zone health). No manual refresh needed. If WebSocket is disconnected: AIStatusCard shows last-known data with stale timestamp in `--color-stale`.

### Threshold-crossing re-inference (invisible to user)
When a zone threshold is crossed, the inference service triggers immediate re-inference for that zone. If the new inference produces a recommendation, it enters the queue via existing WebSocket delta. No specific UI contract — the existing RecommendationCard flow handles it.

### Model hot-reload (invisible to user)
When a new `.onnx` file passes validation, the inference service reloads it without restarting. If the user is on the Home tab, the AIStatusCard maturity data may update on the next WebSocket push. No user notification needed.

## Accessibility Contract

- AIStatusCard progress bars: `role="progressbar"`, `aria-valuenow={weeks_collected}`, `aria-valuemin="0"`, `aria-valuemax="4"`, `aria-label="{domain} data maturity: {N} of 4 weeks"`

- DomainMaturityRow mode badge: `aria-label="{domain} is using {mode} recommendations"`
- Source badge on RecommendationCard: `aria-label="Generated by {source}"` where source is "AI model" or "rule engine"
- Settings icon button in app header: `aria-label="Open AI settings"`
- All touch targets >= 44px height (carried forward)
- Focus ring: 2px solid `--color-accent`, 2px offset (carried forward from prior phases)
