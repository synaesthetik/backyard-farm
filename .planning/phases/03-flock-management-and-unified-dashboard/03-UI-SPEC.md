# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Flock Management and Unified Dashboard.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** This contract covers the Phase 3 delivery: four-tab Home/Zones/Coop/Recs layout,
> new Home tab as unified overview (compact zone cards + flock summary card), egg count
> display driven by nesting box weight sensor, animated "hen present" indicator, 30-day
> production trend chart (actual vs expected, uPlot two-series overlay), production drop alert
> integration, feed consumption rate display with 7-day inline SVG sparkline, feed consumption
> drop alert, and flock configuration settings page.
>
> This contract EXTENDS Phase 2's approved design system (02-UI-SPEC.md). All Phase 1 and
> Phase 2 tokens, typography, and color decisions carry forward unchanged. This document
> specifies only what is new or extended in Phase 3.

## Typography

Identical to Phase 2 — no new type scales introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Zone name heading on ZoneCard; sensor value numerics |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings ("Flock", "Egg Production", "Feed", page titles) |
| Body | 16px | 400 (regular) | 1.5 | Card descriptions, status text, button labels, settings field values |
| Label | 14px | 400 (regular) | 1.4 | Supporting values, timestamps, settings field labels, badge text, sparkline adjacent rate label |

**Constraints (carry-forward):**
- Exactly 2 weights: 400 and 600. No other weights. Merriweather 700 is loaded for system
  CSS defaults only — do not use weight 700 in component CSS.
- Minimum font size: 14px. Nothing smaller.
- Sensor numerics: `font-variant-numeric: tabular-nums` on all updating numeric values
  (egg count, feed rate, production chart axis ticks).
- Italic reserved for zero/placeholder values only ("--").

**New in Phase 3:**
- Production trend indicator arrow (up/down/flat): rendered as a Lucide icon (`TrendingUp`,
  `TrendingDown`, `Minus`) at 16px inline with a Label-role (14px/400) percentage value.
  The icon itself has no text label.
- Refresh button label: Body role (16px/400). Label: "Refresh" (single word — context is
  clear from position on Coop tab).
- Flock settings page heading: Heading role (20px/600). "Flock Settings".
- Settings gear icon link: icon-only button (24px Lucide `Settings` icon). `aria-label="Flock settings"`.
- Feed rate label: Label role (14px/400). Format: "~{N}g/day".
- Hen present indicator: Body role (16px/600) in `--color-accent`. Label: "Hen present".
  Icon: Lucide `Egg` at 16px.

**Canvas exception — uPlot axis tick labels (carry-forward from Phase 2):**
ProductionChart.svelte uses uPlot. Axis tick labels: `size: 14` in uPlot `axes` config.
Do not use sub-14px values in uPlot config. Applies to both the x-axis (dates) and y-axis
(egg count) of the production overlay chart.

### New: HenPresentIndicator

Animated indicator shown on CoopPanel when a hen is detected in the nesting box.

**Visibility:** Rendered only when `hen_present = true` from WebSocket state. When `hen_present = false`, the component is not rendered (not hidden with CSS — use Svelte `{#if}`).

**Layout:** Inline flex row. Align-items: center. Gap: 4px (xs).

**Contents:**
- Lucide `Egg` icon at 16px, `--color-accent`.
- Label: "Hen present" (Body role, 16px/600, `--color-accent`).

**Animation:** The `Egg` icon pulses on a 2-second ease-in-out loop when active.

```css
/* CSS animation — applies only to the icon element when hen_present = true */
@keyframes hen-pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50%       { transform: scale(1.15); opacity: 0.7; }
}
```

**Accessibility:** `aria-live="polite"` on the container. When the indicator appears, screen
readers announce "Hen present". `aria-hidden="true"` on the icon (text carries the meaning).

### New: ProductionChart

uPlot two-series overlay chart. Actual vs expected daily egg count over 30 days.

**Library:** uPlot 1.6.32 — same direct DOM instantiation pattern as SensorChart.svelte
(`onMount` + `ResizeObserver` + `requestAnimationFrame`). No new library.

**Layout:** Full-width of CoopPanel container. Fixed height: 160px on mobile, 200px on
desktop (>= 640px). Background: `--color-bg` applied via uPlot `over` background.

**No time range toggle.** The production chart always shows 30 days. There is no 7-day
view. Omit the toggle buttons present on SensorChart.

**Series configuration:**

| Series | Stroke | Width | Dash | Label |
|--------|--------|-------|------|-------|
| Actual eggs | `#4ade80` (`--color-accent`) | 1.5px | none (solid) | "Actual" |
| Expected eggs | `#94a3b8` (`--color-text-secondary`) | 1.5px | `[4, 4]` | "Expected" |

**uPlot theme (identical to SensorChart except series):**
- Grid lines: `--color-border` (`#2d3149`)
- Axis labels: `--color-text-secondary` (`#94a3b8`)
- Axis tick labels: `size: 14` in uPlot `axes` config (Label role minimum — no sub-14px)
- Cursor crosshair: `--color-muted` (`#6b7280`)
- No area fill under either series — stroke only

**Loading state:** Skeleton — same dimensions as chart, background `--color-surface`,
no animation. Replaced by chart when data is loaded.

**Empty state (fewer than 3 days of egg data):** Skeleton dimensions + centered text:
"Not enough data yet — check back in a few days" (Label, 14px/400, `--color-muted`).

**uPlot instantiation note:** Follow the `SensorChart.svelte` pattern exactly. If
ProductionChart is rendered inside CoopPanel when the Coop tab is not active, defer
instantiation until the tab becomes visible using `{#if}` conditional — not CSS
`visibility: hidden`. See RESEARCH.md Pitfall 2.

### New: FeedSparkline

Inline SVG 7-day sparkline. Renders beside the feed consumption rate label on CoopPanel.

**Dimensions:** 80px wide × 24px tall. Fixed — not responsive.

**Visual:** Single `<polyline>` element. No fill. Stroke: `--color-accent` (`#4ade80`),
stroke-width: 1.5px. `aria-hidden="true"` — decorative only, the rate label carries meaning.

**Empty state:** If fewer than 2 data points exist, render nothing (`{#if}`). Do not render
a flat line or placeholder.

**Data source:** 7 daily consumption values from `dashboardStore.feedConsumption.weekly`
(oldest first). Values normalized to SVG coordinate space in `toPolylinePoints()` using
min/max scaling. Division-by-zero guard: if all values are equal, use range = 1.

### New: FlockSummaryCard

Compact flock summary for the Home tab. Taps through to Coop tab.

**Layout:** Full-width card. Background `--color-surface`. Border: 1px solid `--color-border`.

**Tappable:** Entire card navigates to `/coop` via SvelteKit `goto()`. Add `role="button"`,

**Focal point:** The egg count today is the visual anchor — rendered in Display role

**Accessibility:** `aria-label="Flock summary — tap to see coop detail"` on the card element.

**Empty state:** If no flock data yet (null WebSocket state): render card with "--" in the
egg count position and omit trend indicator. Do not remove the card from the Home tab.

Settings form accessible from `/coop/settings/` route.

**Access:** Lucide `Settings` icon button (24px) in the Coop page header row, right-aligned.

**Layout:** Single-column form. Background `--color-bg`. Max-width: 480px, centered on

**Page heading:** "Flock Settings" (Heading role, 20px/600).

**Form fields (top to bottom):**

| Field | Type | Label | Notes |
|-------|------|-------|-------|
| Breed | Select | "Breed" | Dropdown with pre-populated breed list + "Custom" option |
| Custom lay rate | Number input | "Lay rate (eggs/hen/day)" | Shown only when breed = "Custom"; hidden otherwise |
| Hatch date | Date input | "Hatch date" | ISO date picker; hint: "Used to calculate age factor" |
| Flock size | Number input | "Flock size (hens)" | Integer; min 1 |

| Tare weight | Number input | "Nesting box tare (grams)" | Default 0; hint: "Weigh the empty box and enter here"; alongside a "Set from sensor" button |

| Egg weight | Number input | "Egg weight (grams)" | Default 60; hint: "Used to estimate egg count from box weight" |

disappearing is the confirmation. If the API returns an error, show the error inline at the
bottom of the form above the Save button.

## State Definitions

Phase 3 adds these interaction states to the Phase 2 set.

### Egg Count / Nesting Box States

| State | Trigger | Visual |
|-------|---------|--------|
| No data yet | Null WebSocket state | "--" in Display role, `--color-muted` |
| Count known, no hen | `hen_present = false` | "{N} eggs" in Display role; HenPresentIndicator hidden |
| Count known, hen present | `hen_present = true` | "{N} eggs" in Display role; HenPresentIndicator visible with pulse animation |
| Refresh in-flight | Refresh button tapped | RefreshCw icon spins; button disabled; previous count remains visible |

### Production Chart States

| State | Visual |
|-------|--------|
| Loading | Skeleton at chart dimensions, `--color-surface` background, no animation |
| Fewer than 3 days of data | Skeleton + "Not enough data yet — check back in a few days" centered |
| Data loaded | uPlot chart with actual (solid green) and expected (dashed secondary) series |

### Feed Consumption States

| State | Visual |
|-------|--------|
| No data | Rate label hidden; sparkline hidden |
| Data loaded | "~{N}g/day" label + 7-point sparkline |
| Refill day in window | Sparkline renders normally; refill-day data point is a null gap (no point) |

### Flock Settings Form States

| State | Visual |
|-------|--------|
| Initial load | Form fields populated with current config from API |
| Loading config | Skeleton inputs; Save button disabled |
| Editing | Normal form; no auto-save |
| Save in-flight | Save button shows spinner; all fields disabled |
| Save success | Navigate to `/coop` |
| Save error | Error message inline above Save button; all fields re-enabled |
| "Set from sensor" in-flight | Tare field spinner; "Set from current reading" button disabled |

## Real-Time Behavior Contract

Extends Phase 2 WebSocket contract. Phase 3 adds the following delta message types:

| Event type | UI action |
|-----------|-----------|
| `nesting_box` | CoopPanel egg count updates; HenPresentIndicator toggles; FlockSummaryCard egg count on Home tab updates |
| `feed_consumption` | FeedSparkline updates; rate label updates on CoopPanel |
| `alert_state` | AlertBar re-renders (existing handler) — now includes `production_drop` and `feed_consumption_drop` alert types |

**No optimistic updates (carry-forward from Phase 2):** All state changes on confirmed hub
data only. Refresh button does not optimistically update the egg count.

**WebSocket full snapshot on reconnect (carry-forward):** Server sends full state snapshot
including new `eggCount` and `feedConsumption` fields before streaming deltas.

**Tone (carry-forward, reinforced):** Factual, terse, zero decoration. No emojis. No
exclamation marks. No "Great news!" This dashboard is used while actively managing
farm tasks. Every word earns its place.

**Number format:** Egg counts are integers — no decimal places. Feed rates are integers
rounded to nearest gram (e.g., "~120g/day", not "~120.3g/day"). Trend percentages: one
decimal place is allowed if meaningful (e.g., "87.5%") but integer is preferred when
the decimal is zero.

**Time format:** 24-hour clock throughout (carry-forward).

## Accessibility Minimums

Carry-forward from Phases 1 and 2 plus Phase 3 additions:

- HenPresentIndicator: `aria-live="polite"` on container; `aria-hidden="true"` on icon;
  text "Hen present" carries the semantic meaning.
- ProductionChart: `aria-label="Egg production chart — actual vs expected, last 30 days"` on
  the container div. uPlot canvas is inherently not screen-reader-friendly; the label
  provides context.
- FeedSparkline: `aria-hidden="true"` — the rate label ("~{N}g/day") carries the meaning.
- FlockSummaryCard: `role="button"`, `tabindex="0"`, `aria-label="Flock summary — tap to see
  coop detail"`. Keyboard operable: Enter and Space navigate to `/coop`.
- ZoneCard compact variant: same accessibility as full ZoneCard (carry-forward).
- FlockSettings form: each input has `<label>` with `for` attribute. Hint text uses
  `aria-describedby`. Validation errors use `aria-describedby` on the input and
  `role="alert"` on the error message element.
- Refresh button: `aria-label="Refresh egg count"`. In-flight: `aria-disabled="true"`,
  `aria-busy="true"`.
- Settings gear icon button: `aria-label="Flock settings"`.
- Trend indicator icons: `aria-hidden="true"` — the adjacent percentage value and color
  together convey the trend; icons are decorative in this context.
- Color + text rule (carry-forward): all status states use both color AND text/icon — never
  color alone. Production trend: icon + percentage + color. Alert bar: color + text.
