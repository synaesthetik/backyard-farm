    **API flock router (hub/api/flock_router.py):** New FastAPI APIRouter. Follow the actuator_router.py pattern.
    - `GET /config` — return current flock_config row from DB (or defaults if none). Use the shared db_pool from app state.
    - `PUT /config` — validate and upsert flock_config. Validate: flock_size >= 1, egg_weight_grams > 0, hen_weight_threshold_grams > 0. Return 422 on invalid with detail message. On success, return updated config. Bridge will pick up changes on next periodic loop reload — simpler and acceptable for rarely-changed config.
    - `GET /egg-history?days=30` — return array of {date, actual_count, expected_count} for the last N days. actual_count from egg_counts table. expected_count computed on-the-fly from flock_config using compute_expected_production for each date's daylight factor. Use parameterized queries ($1 placeholders).
    - `POST /refresh-eggs` — trigger an immediate egg count estimation (per D-03 refresh button). Query latest nesting_box_weight reading from sensor_readings, call estimate_egg_count, upsert into egg_counts, return result with estimated_count, hen_present, raw_weight_grams. If no recent reading found, return 404 with detail "No recent nesting box reading available".

    **Mount router (hub/api/main.py):** Add `from . import flock_router` and `app.include_router(flock_router.router, prefix="/api/flock")`.

    **NotifyPayload extension (hub/api/models.py):** Add optional fields for new delta types: estimated_count (int | None), hen_present (bool | None), raw_weight_grams (float | None), rate_grams_per_day (float | None), weekly (list[float] | None).

    **WebSocket manager snapshot extension (hub/api/ws_manager.py):** Add egg_count, feed_consumption fields to the snapshot state cache dictionary. On receiving a nesting_box delta type, update egg_count in cache with {today, hen_present, raw_weight_grams, updated_at}. On receiving a feed_consumption delta type, update feed_consumption in cache with {rate_grams_per_day, weekly}. Include both in the snapshot dict sent on new WebSocket connect. Initialize both to null/None in the default snapshot.
