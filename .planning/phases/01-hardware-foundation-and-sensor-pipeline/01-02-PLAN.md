<threat_model>
## Threat Model — MQTT Authentication and ACL

**Assets:** Sensor data stream, actuator command channel (Phase 2), broker availability.

**Trust boundary:** Each edge node is a trust boundary — a compromised node should not be able to publish to other nodes' topics or subscribe to actuator commands.

**Threats:**

| Threat | Category | Severity | Mitigation |
|--------|----------|----------|------------|
| Unauthorized MQTT publish | Spoofing | HIGH | Per-node username/password + ACL restricting write to own topic prefix only |
| Node impersonation | Spoofing | HIGH | Each node has unique credentials; ACL enforces topic prefix matches username |
| Eavesdropping on sensor data | Information Disclosure | LOW | LAN-only; data is not sensitive (soil moisture readings). Acceptable for Phase 1. |
| Broker denial of service | Denial of Service | LOW | Single-user LAN; max_connections limit in mosquitto.conf |

**HIGH severity mitigations:** Both addressed in this plan via password file + ACL file.
</threat_model>

**docs/mqtt-topic-schema.md:**

Document the complete MQTT topic hierarchy:

```
Topic Schema
============

Sensor readings:
  farm/{node_id}/sensors/{sensor_type}
  Examples:
    farm/zone-01/sensors/moisture    — soil moisture VWC %
    farm/zone-01/sensors/ph          — pH reading
    farm/zone-01/sensors/temperature — temperature in Celsius
    farm/coop/sensors/feed_weight    — feed hopper weight in grams
    farm/coop/sensors/water_level    — water level sensor

Heartbeats:
  farm/{node_id}/heartbeat
  Examples:
    farm/zone-01/heartbeat
    farm/coop/heartbeat

Payload format (JSON, all sensor topics):
{
  "zone_id": "zone-01",
  "sensor_type": "moisture",
  "value": 42.3,
  "ts": "2026-04-07T12:00:00Z",
  "node_id": "zone-01"
}

Heartbeat payload (JSON):
{
  "node_id": "zone-01",
  "ts": "2026-04-07T12:00:00Z",
  "uptime_seconds": 3600
}

QoS: 1 (at least once delivery) for all publishes.
Retain: false for sensor readings, true for heartbeats (last known state).

Node credentials:
  Each node authenticates with username = node_id (e.g., "zone-01").
  Hub bridge authenticates as "hub-bridge" with read/write to all farm/# topics.

Reserved topic prefixes for Phase 2:
  farm/{node_id}/commands/{command_type} — actuator commands (hub to edge)
  farm/{node_id}/ack/{command_id}       — command acknowledgments (edge to hub)
```

**hub/mosquitto/mosquitto.conf:**
```
# Mosquitto 2.x configuration for farm MQTT broker
listener 1883
protocol mqtt

# Authentication — Mosquitto 2.x requires explicit config
allow_anonymous false
password_file /mosquitto/config/passwd
acl_file /mosquitto/config/acl

# Persistence
persistence true
persistence_location /mosquitto/data/

# Logging
log_dest stdout
log_type error
log_type warning
log_type notice
connection_messages true

# Limits
max_connections 50
max_inflight_messages 20
max_queued_messages 1000
```

**hub/mosquitto/acl:**
```
# Hub bridge — superuser access to all farm topics
user hub-bridge
topic readwrite farm/#

# Zone nodes — write own sensors and heartbeat only
user zone-01
topic write farm/zone-01/sensors/#
topic write farm/zone-01/heartbeat

user zone-02
topic write farm/zone-02/sensors/#
topic write farm/zone-02/heartbeat

user zone-03
topic write farm/zone-03/sensors/#
topic write farm/zone-03/heartbeat

user zone-04
topic write farm/zone-04/sensors/#
topic write farm/zone-04/heartbeat

# Coop node — write own sensors and heartbeat only
user coop
topic write farm/coop/sensors/#
topic write farm/coop/heartbeat

# Deny all other access by default (Mosquitto 2.x behavior)
```

**hub/mosquitto/generate-passwords.sh:**
```bash
#!/usr/bin/env bash
# Generate Mosquitto password file for all farm nodes.
# Run from hub/ directory: bash mosquitto/generate-passwords.sh
# Passwords are printed to stdout — save them for edge node configuration.

set -euo pipefail

PASSWD_FILE="mosquitto/passwd"

# Remove existing file to start fresh
rm -f "$PASSWD_FILE"

echo "Generating MQTT credentials..."
echo "================================"

# Hub bridge (superuser)
BRIDGE_PASS=$(openssl rand -base64 16)
docker compose exec mosquitto mosquitto_passwd -b /mosquitto/config/passwd hub-bridge "$BRIDGE_PASS" 2>/dev/null || \
  mosquitto_passwd -c -b "$PASSWD_FILE" hub-bridge "$BRIDGE_PASS"
echo "hub-bridge: $BRIDGE_PASS"

# Zone nodes
for i in 01 02 03 04; do
  ZONE_PASS=$(openssl rand -base64 16)
  docker compose exec mosquitto mosquitto_passwd -b /mosquitto/config/passwd "zone-$i" "$ZONE_PASS" 2>/dev/null || \
    mosquitto_passwd -b "$PASSWD_FILE" "zone-$i" "$ZONE_PASS"
  echo "zone-$i: $ZONE_PASS"
done

# Coop node
COOP_PASS=$(openssl rand -base64 16)
docker compose exec mosquitto mosquitto_passwd -b /mosquitto/config/passwd coop "$COOP_PASS" 2>/dev/null || \
  mosquitto_passwd -b "$PASSWD_FILE" coop "$COOP_PASS"
echo "coop: $COOP_PASS"

echo "================================"
echo "Password file: $PASSWD_FILE"
echo "Save these credentials for edge node .env files."
```
