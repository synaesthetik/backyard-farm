<threat_model>
## Threat Model (ASVS Level 1)

**Change scope:** Caddy reverse proxy routing for WebSocket path; removal of server.js WS handling.

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Auth bypass by routing WS to different backend | LOW | No auth change — `/ws/dashboard` was already a public endpoint on server.js; routing to api:8000 does not introduce new access. FastAPI ws_manager has same public access model. |
| Internal notify endpoint exposure | LOW (pre-existing) | `/internal/notify` on api:8000 has no authentication — this is a pre-existing condition noted in VERIFICATION.md, NOT introduced by this change. Caddy already routes `/api/*` to api:8000. No new endpoints exposed. |
| WebSocket message injection | LOW | WebSocket is server-to-client broadcast only; client sends no actionable data (ws.receive_text() is just keepalive). |

**Verdict:** No new threats introduced. This change reduces attack surface by removing the server.js WebSocket handler (one fewer WebSocket endpoint).
</threat_model>
