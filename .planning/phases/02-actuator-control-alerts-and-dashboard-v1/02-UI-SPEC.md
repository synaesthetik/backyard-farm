# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Actuator Control, Alerts, and Dashboard V1.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** This contract covers the Phase 2 delivery: multi-route SPA with bottom tab
> bar, persistent P0/P1 alert bar, irrigation manual control, sensor-feedback irrigation
> loop, coop door automation and manual override, feed/water level display, recommend-
> and-confirm UX with rule-based queue, zone health composite score, 7-day/30-day sensor
> history charts, and PWA service worker + offline shell.
>
> This contract EXTENDS Phase 1's approved design system (01-UI-SPEC.md). All Phase 1
> tokens, typography, and color decisions carry forward unchanged. This document specifies
> only what is new or extended in Phase 2.

## Spacing Scale

Declared values (multiples of 4 only). Identical to Phase 1 — no new tokens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline badge padding, quality badge internal padding |
| sm | 8px | Sensor value row internal spacing, chip/badge padding |
| md | 16px | Zone card internal padding, tab bar internal padding, section padding |
| lg | 24px | Between zone cards in grid, gap between alert bar rows, section heading margin |
| xl | 32px | Page horizontal padding (desktop), section breaks |
| 2xl | 48px | Major section breaks; header height (48px) |
| 3xl | 64px | Not used in Phase 2 — reserved |

Exceptions (carry-forward from Phase 1, extended for Phase 2):
- Touch targets: 44px minimum hit area for all interactive controls — applies to tab bar
  tabs, approve/reject buttons, coop open/close buttons, valve open/close buttons.
- Bottom tab bar height: 56px visual height + env(safe-area-inset-bottom) padding below.
- Alert bar rows: auto-height per row (no fixed height); min-height 40px per alert row.
- Bottom safe area inset: max(16px, env(safe-area-inset-bottom)) applies to page bottom
  padding AND to bottom tab bar bottom padding.
- Toast internal padding: 12px — intermediate value used to avoid visual compression at
  8px (sm) while respecting the card-like density of the toast at 16px (md). Applies only
  to the Toast component padding property.

## Typography

Identical to Phase 1 — no new type scales introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Zone name heading on ZoneCard; sensor value numerics |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings ("Recommendations", "Coop", tab page titles) |
| Body | 16px | 400 (regular) | 1.5 | Recommendation descriptions, status text, button labels, alert bar text |
| Label | 14px | 400 (regular) | 1.4 | Supporting sensor values in recommendation cards, timestamps, secondary copy, badge text |

**Constraints (carry-forward):**
- Exactly 2 weights: 400 and 600. No other weights.
- Minimum font size: 14px. Nothing smaller.
- Sensor numerics: `font-variant-numeric: tabular-nums` on all updating numeric values.
- Italic reserved for zero/placeholder values only ("--").

**New in Phase 2:**
- Command button labels: Body role (16px/600). Action-verb + noun where context warrants
  disambiguation (see Copywriting Contract for exact labels). Do not use label role for
  primary action buttons.
- Tab bar labels: Label role (14px/400). Single-word labels only: "Zones", "Coop",
  "Recommendations" (abbreviated to "Recs" if layout requires).
- Alert bar text: Body role (16px/400). Truncated to single line per alert entry.
- Toast text: Body role (16px/400).

**Canvas exception — uPlot axis tick labels only:**
uPlot axis tick labels are set to `size: 14` in the uPlot `axes` config (mapping to the
Label role). Do NOT use a sub-14px value. Although 12px is a common default in charting
libraries, the minimum font size constraint of 14px applies here — axis ticks are read
while standing outdoors in variable light conditions and must remain legible. This is a
uPlot canvas config value, not a CSS `font-size` declaration.

### Phase 1 Components — Phase 2 Extensions

**ZoneCard** — additions:
- Tappable: tap navigates to `/zones/[id]` via SvelteKit `goto()`. Add `role="button"`,
  `tabindex="0"`, `cursor: pointer`.

### New: TabBar

Persistent bottom navigation. Rendered in `+layout.svelte`.

**Layout:** Fixed bottom, full-width. Height: 56px + env(safe-area-inset-bottom).
Background: `--color-surface`. Top border: 1px solid `--color-border`.
z-index: 20 (above page content, below toasts).

**Three tabs:**

| Tab | Icon (Lucide) | Label | Route |
|-----|--------------|-------|-------|
| Zones | `Sprout` | "Zones" | `/` |
| Coop | `Home` | "Coop" | `/coop` |
| Recommendations | `Bell` | "Recs" | `/recommendations` |

**Active tab indicator:** `--color-accent` underline (2px solid, bottom edge of tab).
Active tab label and icon: `--color-accent`. Inactive: `--color-muted`.

**Touch target:** Each tab occupies 1/3 of the bar width. 44px minimum tap area height
enforced via min-height on the tab button element.

**Accessibility:** `<nav aria-label="Main navigation">` wrapper. Each tab is `<a>` with
`aria-current="page"` when active. No JS navigation needed — SvelteKit handles routing.

---

### New: AlertBar

Persistent alert display. Positioned below header (48px), above page content.
Part of `+layout.svelte` layout stack.

**Layout:** Full-width. Auto-height — one row per active alert type. No collapse animation
(D-10 context). Background: `--color-surface`. Bottom border: 1px solid `--color-border`.

**Alert row layout:** Horizontal flex. Left: severity indicator (4px wide solid bar using
alert color) + icon (16px) + alert text (Body, 16px/400). Right: count badge if grouped.
Row padding: 8px vertical (sm), 16px horizontal (md). Min-height: 40px.

**P0 row:** 4px left border in `--color-offline`. `AlertTriangle` Lucide icon (16px), red.
**P1 row:** 4px left border in `--color-stale`. `AlertTriangle` Lucide icon (16px), amber.

**Tappable (D-04):** Entire row is tappable. `role="button"`, `tabindex="0"`. Navigates
via SvelteKit `goto()` to the deep-link target for that alert type.

**Grouping (D-12):** Multiple alerts of the same type render as one row with count badge.
Badge: pill, 12px/400, `--color-border` background, `--color-text-primary` text. Example:
"Low moisture — 3 zones".

**Empty state:** AlertBar is not rendered when there are no active alerts (zero height).
This is not a visual empty state — the component simply unmounts.

**ARIA:** `role="alert"` on the AlertBar container. `aria-live="assertive"` for P0 entries.
`aria-live="polite"` for P1 entries.

---

### New: HealthBadge

Compact health score indicator for ZoneCard header.

**Layout:** Pill badge, inline. 12px/400. 4px horizontal padding (xs), 2px vertical.
Background color per health score mapping. Text: "GOOD" / "WARN" / "CRIT".

**Text color:** Dark (`--color-bg`, `#0f1117`) on green and yellow backgrounds.
Light (`--color-text-primary`) on red background for contrast.

**Accessibility:** `aria-label="Zone health: {score}"` on the badge element.

---

### New: CommandButton

Wrapper component for actuator command buttons (irrigation and coop door). Encapsulates
the spinner + disabled state pattern (D-14).

**States:**

| State | Visual |
|-------|--------|
| Idle | Normal button — background `--color-surface`, border `--color-border`, text `--color-text-primary` |
| In-flight | Spinner (animated, 16px, `--color-accent`) replaces button text; button disabled; opacity 0.6 |
| Error | Re-enabled automatically after toast shown (D-16) |

**Approve variant:** Background `--color-accent` (`#4ade80`), text `--color-bg` (dark on
green). Used only for the Approve control on RecommendationCard.

**Reject variant:** Background `--color-surface`, border `--color-border`, text
`--color-text-primary`. No red.

**Open/Close valve variant:** Background `--color-surface`, border `--color-border`.
"Open valve" and "Close valve" labeled buttons side by side. Active state (valve currently
open): "Open valve" button has `--color-accent` border.

**Open/Close door variant:** Background `--color-surface`, border `--color-border`.
"Open door" and "Close door" labeled buttons side by side. Door states: if `moving`, both
buttons disabled with spinner on the active one.

**Touch target:** 44px minimum height on all CommandButton instances.

**Spinner:** CSS animation, rotating `Loader2` Lucide icon at 16px, `--color-accent`.
No third-party spinner library.

---

### New: Toast

Fixed-position error notification. Appears on command failure (D-16).

**Layout:** Fixed, bottom center, above tab bar. `bottom: calc(56px + env(safe-area-inset-bottom) + 8px)`.
Width: min(calc(100% - 32px), 400px). Centered horizontally. z-index: 30 (above everything).

**Visual:** Background `--color-offline` (`#ef4444`), text `--color-text-primary`.
Border-radius: 8px. Padding: 12px (named exception — see Spacing Scale exceptions). Body
role (16px/400). `XCircle` Lucide icon (16px) at left.

**Auto-dismiss:** 5 seconds after display. No dismiss button needed.

**Content:** Single-line text. Two fixed messages in Phase 2:
- "Command failed — tap to retry" — for generic ack timeout
- "Another zone is already irrigating." — for single-zone invariant rejection (D-17)

**Accessibility:** `role="alert"`, `aria-live="assertive"`.

---

### New: RecommendationCard

Displays one pending recommendation in the queue.

**Layout:** Full-width card. Background `--color-surface`. Border: 1px solid `--color-border`.
Border-radius: 8px. Internal padding: 16px (md).

**Focal point:** The Approve button on the first pending RecommendationCard is the primary
visual anchor for the `/recommendations` route. It is rendered in `--color-accent` (green
on dark) to draw the eye immediately. When the queue is empty, the empty-state heading
"No recommendations" serves as the focal point (centered, Heading role).

**Contents (top to bottom):**
1. Action description — Body role (16px/400), `--color-text-primary`. Example: "Irrigate Zone A"
2. Supporting sensor reading — Label role (14px/400), `--color-text-secondary`.
   Format: "Moisture: 18% VWC (target range: 40–60%)"
3. Explanation — Label role (14px/400), `--color-text-secondary`, italic.
   Example: "Below low threshold for 2h"
4. Controls row (bottom): Approve button (CommandButton approve variant) + Reject button
   (CommandButton reject variant). Justify: space-between. Gap: 8px (sm).

**Empty state (no pending recommendations):**
- Heading: "No recommendations" (Heading role, 20px/600)
- Body: "Sensor thresholds are within range." (Body role, 16px/400, `--color-muted`)

**Deduplication:** UI renders what the hub provides. If a duplicate is suppressed hub-side,
it does not appear. No UI-side dedup logic needed.

---

### New: CoopPanel

The full content area of the `/coop` route.

**Layout:** Single-column, full-width. Sections stack vertically with 24px gap (lg).

**Sections (top to bottom):**

1. **Door status section:**
   - Section heading: "Coop Door" (Heading, 20px/600)
   - State display: Large state label (Body, 16px/400, colored per door state mapping).
     Door icon: `DoorOpen` (open/moving) or `DoorClosed` (closed/stuck) Lucide, 24px.
   - Control row: "Open door" CommandButton + "Close door" CommandButton side by side.

2. **Schedule display (read-only):**
   - Label: "Today's schedule" (Label, 14px/400, `--color-text-secondary`)
   - Next open time + Next close time in Label role (14px/400).
   - No edit controls — schedule is computed from lat/long env vars.

3. **Feed level section:**
   - Section heading: "Feed" (Heading, 20px/600)
   - Fill percentage displayed as a horizontal progress bar + numeric label.
   - Progress bar: height 8px, `--color-border` background, `--color-accent` fill.
   - When below threshold: fill changes to `--color-stale` (amber warning).
   - Numeric label: "73%" in Body role (16px/400) beside the bar.

4. **Water level section:**
   - Same layout as Feed level. Section heading: "Water".
   - Progress bar fill: `#60a5fa` (same blue used for pH in charts) when above threshold.
   - When below threshold: fill changes to `--color-stale`.

### New: SensorChart

Time-series chart wrapping uPlot. One chart instance per sensor type per zone detail page.

**Library:** uPlot (direct instantiation in `onMount`/`onDestroy`, no Svelte wrapper
package). ~45KB canvas-based.

**Layout:** Full-width of container. Fixed height: 160px on mobile, 200px on desktop
(>= 640px). Background: `--color-bg` (`#0f1117`) applied via uPlot `over` background.

**Time range toggle:** Two buttons above the chart: "7 days" | "30 days". Label role
(14px/400). Active selection: `--color-accent` text + underline. Inactive: `--color-muted`.
Toggle switches without page reload (D-32 — re-fetches from REST endpoint).

**Series configuration per sensor type:**

| Sensor | Series color | Label |
|--------|-------------|-------|
| Moisture | `#4ade80` (`--color-accent`) | "VWC %" |
| pH | `#60a5fa` | "pH" |
| Temperature | `#fb923c` | "°C" |

**uPlot theme overrides:**
- Grid lines: `--color-border` (`#2d3149`)
- Axis labels: `--color-text-secondary` (`#94a3b8`)
- Axis values (tick labels): `--color-text-secondary`, `size: 14` in uPlot axes config
  (Label role — see Typography canvas exception; do not use sub-14px values)
- Series stroke width: 1.5px
- No area fill under lines — stroke only
- Cursor crosshair: `--color-muted` (`#6b7280`)

**Loading state:** Skeleton placeholder — same dimensions as chart, background
`--color-surface`, no animation. Replaced by chart on data load.

**Empty state (no data for range):** Show skeleton dimensions + centered text:
"No data for this period" (Label, 14px/400, `--color-muted`).

## State Definitions

Phase 2 adds these interaction states to the Phase 1 set.

### Command Button States

| State | Trigger | Visual |
|-------|---------|--------|
| Idle | No command in flight | Normal button |
| In-flight | Command sent, awaiting ack | Spinner, disabled |
| Confirmed | Ack received | Returns to idle; status update reflects in display |
| Failed | Hub timeout (server-side, 10s) | Toast shown, button re-enabled |

### Coop Door States (D-24)

| State | Color | Label | Icon |
|-------|-------|-------|------|
| Open | `--color-accent` | "OPEN" | `DoorOpen` |
| Closed | `--color-text-secondary` | "CLOSED" | `DoorClosed` |
| Moving | `--color-stale` | "MOVING" | `DoorOpen` with spinner adjacent |
| Stuck | `--color-offline` | "STUCK" | `AlertTriangle` |

### Recommendation States

| State | Display |
|-------|---------|
| Pending | RecommendationCard in queue |
| Approved (in-flight) | Approve button shows spinner; card remains visible |
| Approved (confirmed) | Card removed from queue; no success toast |
| Rejected | Card removed immediately from UI; back-off managed hub-side |

### Zone Detail — Irrigation States

| State | Display |
|-------|---------|
| Valve closed | "Closed" label; "Open valve" button active, "Close valve" button disabled |
| Valve open (irrigating) | "Irrigating" label with `--color-accent`; "Close valve" button active, "Open valve" button disabled; session timer visible |
| Valve moving/transitioning | Both buttons disabled; spinner |
| Error | Error toast; both buttons re-enabled |

## Real-Time Behavior Contract

**Extends Phase 1 WebSocket contract.** Phase 2 adds the following to the WebSocket delta
message types (all rendered from hub-pushed state, no client-side inference):

| Event type | UI action |
|-----------|-----------|
| `alert_state` | AlertBar re-renders with current alert list |
| `recommendation_queue` | RecommendationCard list updates |
| `zone_health_score` | HealthBadge on ZoneCard updates |
| `actuator_state` (irrigation) | Irrigation status on zone detail updates; CommandButton state updates |
| `actuator_state` (coop door) | Door state label on CoopPanel updates; CommandButton state updates |
| `feed_level` | Feed progress bar on CoopPanel updates |
| `water_level` | Water progress bar on CoopPanel updates |

**No optimistic updates (D-15).** Status only changes on confirmed hub ack. During
in-flight period, the previous state is shown (dimmed) and the button is disabled.

**WebSocket full snapshot on reconnect:** Same as Phase 1 — server sends full state
snapshot on connect/reconnect before streaming deltas.

## PWA Contract

Phase 2 delivers the service worker and offline shell (D-33, UI-05).

**Service worker (SvelteKit built-in):**
- Pre-cache: app shell (HTML, CSS, JS, manifest, icons)
- Do NOT cache data endpoints (`/api/*`, `/ws/*`)
- Stale-while-revalidate strategy for app shell assets
- Offline fallback: render app shell with stale sensor values (freshness system already
  handles stale display correctly from Phase 1)

**Manifest updates (additions to Phase 1 manifest):**
- `start_url: "/"` — ensures install opens to Zones tab
- `display: "standalone"` — carry-forward
- `orientation: "portrait"` — primary use case is standing in the yard, one hand

**iOS safe areas:** `env(safe-area-inset-bottom)` applied to tab bar bottom padding and
page bottom padding. `viewport-fit=cover` in meta viewport (Phase 1, carry-forward).

**Tone (carry-forward, reinforced):** Factual, terse, zero decoration. No emojis. No
exclamation marks. No "Everything looks great!" This is used while actively managing
farm tasks. Every word earns its place.

**Time format:** 24-hour clock throughout. "18:32", not "6:32 PM".

**Duration format:** "{N}m {N}s" for short durations (< 1 hour). "{N}h {N}m" for longer.
Do not write "minutes" or "seconds" — abbreviate.

## Accessibility Minimums

Carry-forward from Phase 1 plus Phase 2 additions:

- All interactive controls (CommandButton, tab bar, recommendation controls) must be
  operable by keyboard (Tab + Enter/Space).
- CommandButton in-flight state: `aria-disabled="true"` + `aria-busy="true"`.
- AlertBar: `role="alert"`, `aria-live="assertive"` for P0, `aria-live="polite"` for P1.
- TabBar: `<nav aria-label="Main navigation">`, active tab `aria-current="page"`.
- HealthBadge: `aria-label="Zone health: {Good | Warning | Critical}"`.
- RecommendationCard approve/reject: Buttons labeled with `aria-label="Approve: Irrigate
  {Zone name}"` and `aria-label="Reject: Irrigate {Zone name}"` to disambiguate when
  multiple cards are present.
- Coop door state label: `aria-live="polite"` so state transitions are announced.
- Toast: `role="alert"`, `aria-live="assertive"`.
- Color + text: all status states use both color AND text/icon — never color alone.
