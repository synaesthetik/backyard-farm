| Dominant (60%) | `--color-bg` | `#0f1117` | Page background |

**Accent (`#4ade80`) reserved exclusively for:**

- Header title text (inherited)

**Accent must NOT be used for:** destructive actions, info-only labels, storage stat numbers, disabled states.

**P1 amber (`#f59e0b`) used for:**
- pH calibration overdue alert severity bar in AlertBar (same as existing stale/P1 pattern)
- "Overdue" badge on calibration management table row

## Component Inventory

New components introduced in Phase 5 (follow established Svelte 5 / CSS variable patterns):

### CalibrationStatusBadge
- Pill badge: text "Due in N days" or "OVERDUE"
- Overdue state: background `var(--color-stale)`, text `#0f1117`, 14px/400
- Due soon (< 3 days): same amber color
- OK state: background `var(--color-border)`, text `var(--color-text-secondary)`, 14px/400
- Border-radius: 4px (matches existing quality-badge)
- Visual differentiation from surrounding text is provided by background color and border-radius — no smaller font size is needed

### CalibrationRow (used in /settings/calibration table)
- Layout: flex row, align-items center, `min-height: 44px`
- Left: zone name 16px/400, sensor type label 14px/400 muted
- Right: CalibrationStatusBadge + "Record Calibration" button
- "Record Calibration" button: outline style (border `1px solid var(--color-accent)`, text `var(--color-accent)`) when calibration not overdue; filled style (background `var(--color-accent)`, text `#0f1117`) when overdue
- Divider: `border-bottom: 1px solid var(--color-border)` between rows

### NtfySettingsForm (used in /settings/notifications)
- Two text inputs: ntfy Server URL, Topic
- Input style: `background: var(--color-bg)`, `border: 1px solid var(--color-border)`, `border-radius: var(--radius-sm)`, `padding: var(--spacing-sm) var(--spacing-md)`, `color: var(--color-text-primary)`, `font-family: Inter`, `font-size: 16px`
- Focus state: `border-color: var(--color-accent)`, `outline: none`
- Enabled toggle: reuse AISettingsToggle pattern (toggle-switch aria-role="switch")
- "Send Test" button: primary style (`background: var(--color-accent)`, `color: #0f1117`, `font-weight: 600`, `border-radius: var(--radius-md)`, `min-height: 44px`, `padding: 0 var(--spacing-lg)`)
- "Send Test" loading state: text becomes "Sending..." + spinner (same pattern as CommandButton)
- Save button: same primary style as "Send Test", label "Save Settings"

### StoragePanel (used in /settings/storage)
- Table of per-table sizes: zone name column 16px/400, size column 28px/600 tabular-nums right-aligned
- Retention policy row: label 14px/400 muted, policy value 16px/400 primary
- Last purge row: label 14px/400 muted, timestamp 14px/400 secondary
- "Purge Now" button: destructive style (`border: 1px solid var(--color-offline)`, `color: var(--color-offline)`, `background: transparent`, `min-height: 44px`)

### PhCalibrationInlineAction (used in /zones/[id] pH sensor row)
- Placed immediately below the pH SensorValue row in the sensors-section
- "Record Calibration" link-button: 14px/400 Inter, `color: var(--color-accent)`, no underline, tap target `min-height: 44px`
- When overdue: accompanied by CalibrationStatusBadge "OVERDUE" inline

## Interaction Contracts

### pH Calibration Alert (AlertBar)
- Alert type: P1 (amber severity bar, `--color-stale`)
- Message text: "pH calibration overdue — {zone_id}" (em dash, not hyphen)
- Deep-link: `/zones/{zone_id}` — tapping routes to the zone detail page
- Follows existing AlertBar.svelte pattern exactly; no new AlertBar code needed

### Record Calibration Action (Zone Detail + Calibration Settings)
- Inline in `/zones/[id]`: "Record Calibration" link-button below pH row
- Full form in `/settings/calibration`: CalibrationRow per sensor
- No modal — calibration is recorded inline with a single button tap (no offset entry required from farmer — offset is computed by hub from pH buffer values)
- On success: Toast message "Calibration recorded for {zone_id} pH sensor", alert clears from AlertBar
- On failure: Toast "Calibration save failed — check connection"

### ntfy Send Test
- Button dispatches POST to `/api/settings/ntfy/test`
- Loading state during in-flight request: button text "Sending...", button disabled
- Success: Toast "Test notification sent — check your phone"
- Failure: Toast "Test failed — verify ntfy URL and topic"
- No modal, no confirmation dialog

### Purge Now (Storage page)
- Button triggers confirmation dialog (inline, not browser `confirm()`)
- Confirmation UI: inline warning panel below the button
  - Panel background: `rgba(239, 68, 68, 0.08)` (destructive tint)
  - Border: `1px solid var(--color-offline)`
  - Text: "This will permanently delete all raw sensor readings older than 90 days. This cannot be undone."
  - Font: 14px/400 Inter, `color: var(--color-text-primary)`
  - Two buttons: "Keep My Data" (ghost style) and "Confirm Purge" (filled destructive: `background: var(--color-offline)`, `color: #f1f5f9`)
- On success: Toast "Purge complete — storage reclaimed"
- On failure: Toast "Purge failed — check logs"

### Settings Save Pattern
- PATCH requests are optimistic where state is simple (toggle)
- Form saves (ntfy URL/topic, calibration offsets) are not optimistic — wait for 200 response, then update
- Error reversal: restore previous field values on failure + Toast error

## Accessibility Contract

- All interactive elements: `min-height: 44px; min-width: 44px` (touch target)
- Toggle switches: `role="switch"`, `aria-checked={boolean}`, `aria-label` describing what is toggled
- Destructive confirmation panel: `role="alert"` on the warning text, rendered inline (not `aria-live` since it appears on interaction)
- Form inputs: each `<input>` has an associated `<label>` with matching `for`/`id`
- AlertBar: existing `role="alert"` and `aria-live` pattern unchanged for pH calibration alert
- Focus visible: `outline: 2px solid var(--color-accent); outline-offset: 2px` — carry from established AISettingsToggle pattern
- Error states on inputs: `border-color: var(--color-offline)` + `aria-describedby` pointing to error message span
- "Send Test" and "Confirm Purge" buttons: `aria-busy="true"` during loading
