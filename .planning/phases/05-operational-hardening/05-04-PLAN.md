  <action>
    This is a human verification checkpoint. No code changes are made.
    The executor presents the verification steps below to the farmer and waits for approval.

    Complete operational hardening features built in Plans 01-03:
    1. pH calibration workflow with overdue alerts, recording, and management
    2. ntfy push notification integration with settings, toggle, and Send Test
    3. Data retention with storage stats and manual Purge Now
    4. Settings navigation connecting all four settings pages

    Start the full stack: `cd hub && docker compose up --build -d`

    **1. Settings Navigation**
    - Visit the dashboard and tap the settings icon in the header
    - Verify sub-navigation shows tabs: AI, Calibration, Notifications, Storage
    - Tap each tab to verify it navigates to the correct page

    **2. Calibration Management (/settings/calibration)**
    - Navigate to /settings/calibration
    - Verify the page lists all pH sensor calibration entries with status badges
    - If calibration entries exist: verify CalibrationStatusBadge shows "OVERDUE" (amber) or "Calibrated N days ago" (muted) correctly
    - If no entries exist: verify empty state "No pH sensors configured..." message
    - Tap "Record Calibration" on any entry: verify toast "Calibration recorded for {zone_id} pH sensor"
    - Verify the badge updates after recording

    **3. Zone Detail Inline Calibration**
    - Navigate to any zone with a pH sensor (e.g., /zones/zone-01)
    - Below the pH SensorValue row, verify "Record Calibration" link-button appears
    - If overdue: verify "OVERDUE" badge appears inline
    - Tap the link-button: verify toast message

    **4. pH Calibration Alert**
    - If any pH sensor has no calibration record or an overdue calibration:
      Verify the AlertBar shows "pH calibration overdue -- {zone_id}" with amber severity bar
    - Tap the alert: verify it navigates to /zones/{zone_id}
    - After recording calibration: verify the alert disappears from AlertBar

    **5. ntfy Settings (/settings/notifications)**
    - Navigate to /settings/notifications
    - Verify page heading "Push Notifications"
    - If ntfy is not configured: verify empty state "Push notifications are off..."
    - Toggle the enabled switch and enter a valid ntfy URL (e.g., https://ntfy.sh) and topic
    - Tap "Save Settings": verify toast "Notification settings saved"
    - Tap "Send Test": verify button shows "Sending..." then toast "Test notification sent -- check your phone"
    - (If ntfy server is accessible: check phone for test notification)

    **6. Storage and Retention (/settings/storage)**
    - Navigate to /settings/storage
    - Verify page heading "Storage and Retention"
    - Verify per-table sizes are displayed with numeric values (e.g., "sensor_readings: 2.3 MB")
    - Verify retention note: "Raw readings older than 90 days are automatically purged. Hourly rollups are retained for 2 years."
    - Tap "Purge Now": verify inline confirmation panel appears with warning text
    - Tap "Keep My Data": verify confirmation dismisses
    - Tap "Purge Now" again then "Confirm Purge": verify toast "Purge complete -- storage reclaimed"

    **7. Mobile Layout**
    - Open browser dev tools, toggle to mobile viewport (375px width)
    - Verify all three settings pages are usable on mobile without horizontal scroll
    - Verify touch targets are at least 44px
