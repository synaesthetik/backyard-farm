    Create `hub/api/storage_router.py`:

    ```python
    """REST endpoints for storage stats and manual purge (D-12).

    Queries TimescaleDB directly (not proxied to bridge) since these are
    read-only DB queries and the API process has its own DB pool.

    Endpoints:
      GET  /api/storage/stats  — per-table sizes, retention policy status
      POST /api/storage/purge  — manual drop_chunks for raw data > 90 days

    Exports: router
    """
    ```

    - `GET /api/storage/stats`: Query DB pool directly (from `get_db_pool()`). Run two queries:
      1. Per-table sizes:
         ```sql
         SELECT
             schemaname,
             tablename,
             pg_size_pretty(pg_total_relation_size(schemaname || '.' || tablename)) AS size,
             pg_total_relation_size(schemaname || '.' || tablename) AS size_bytes
         FROM pg_tables
         WHERE schemaname = 'public'
         ORDER BY pg_total_relation_size(schemaname || '.' || tablename) DESC
         ```
      2. Retention policy info:
         ```sql
         SELECT
             hypertable_name,
             schedule_interval::text,
             config
         FROM timescaledb_information.jobs
         WHERE proc_name = 'policy_retention'
         ```
      Return JSON: `{"tables": [...], "retention_policies": [...], "total_size": "...", "total_bytes": int}`.

    - `POST /api/storage/purge`: Execute `SELECT drop_chunks('sensor_readings', INTERVAL '90 days')`. Return `{"status": "ok", "message": "Purge complete"}`. On error, return 500 with detail.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dashboard -> API | Untrusted input: ntfy URL could be any URL, topic could be any string |
| API -> bridge | Internal network, trusted but validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | ntfy_router.py | mitigate | Pydantic BaseModel validates url/topic/enabled types at API boundary; NtfySettings validates on save |
| T-05-05 | Information Disclosure | send_ntfy_notification | accept | Alert messages sent to farmer's own ntfy server; same info already visible on dashboard; no PII |

| T-05-08 | Denial of Service | /api/storage/purge | accept | Single-user LAN system; farmer controls purge; no rate limiting needed |
