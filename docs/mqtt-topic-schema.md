## Topic Hierarchy

### Sensor Readings

```
farm/{node_id}/sensors/{sensor_type}
```

| Example topic | Description |
|---|---|
| `farm/zone-01/sensors/moisture` | Soil moisture VWC % |
| `farm/zone-01/sensors/ph` | pH reading |
| `farm/zone-01/sensors/temperature` | Temperature in Celsius |
| `farm/coop/sensors/feed_weight` | Feed hopper weight in grams |
| `farm/coop/sensors/water_level` | Water level sensor |
| `farm/coop/sensors/nesting_box_weight` | Nesting box weight in grams (Phase 3) |

## Payload Formats

### Sensor Reading Payload (JSON, all `farm/{node_id}/sensors/{sensor_type}` topics)

```json
{
  "zone_id": "zone-01",
  "sensor_type": "moisture",
  "value": 42.3,
  "ts": "2026-04-07T12:00:00Z",
  "node_id": "zone-01"
}
```

| Field | Type | Description |
|---|---|---|
| `zone_id` | string | Zone identifier (same as `node_id` for zone nodes; `"coop"` for coop node) |
| `sensor_type` | string | Sensor type key: `moisture`, `ph`, `temperature`, `feed_weight`, `water_level` |
| `value` | float | Sensor reading in the sensor's native unit |
| `ts` | string (ISO 8601 UTC) | Timestamp of the reading on the edge node |
| `node_id` | string | Node identifier (matches the `{node_id}` in the topic path) |

### Heartbeat Payload (JSON)

```json
{
  "node_id": "zone-01",
  "ts": "2026-04-07T12:00:00Z",
  "uptime_seconds": 3600
}
```

| Field | Type | Description |
|---|---|---|
| `node_id` | string | Node identifier |
| `ts` | string (ISO 8601 UTC) | Timestamp on the edge node |
| `uptime_seconds` | integer | Node uptime in seconds since last boot |

## Delivery Contract

| Property | Value | Rationale |
|---|---|---|
| QoS | 1 (at least once) | Sensor readings must not be silently dropped; duplicate handling is acceptable |
| Retain (sensor readings) | `false` | Hub always consumes live readings; stale retained values would be misleading |
| Retain (heartbeats) | `true` | Hub needs last-known-alive state after reconnect |

## Node Credentials

Each node authenticates to Mosquitto using its node ID as the username.

| Node | Username | Topic write prefix |
|---|---|---|
| Zone 01 | `zone-01` | `farm/zone-01/` |
| Zone 02 | `zone-02` | `farm/zone-02/` |
| Zone 03 | `zone-03` | `farm/zone-03/` |
| Zone 04 | `zone-04` | `farm/zone-04/` |
| Coop | `coop` | `farm/coop/` |
| Hub bridge | `hub-bridge` | `farm/#` (read/write all) |

Passwords are generated by `hub/mosquitto/generate-passwords.sh`. Each call generates a fresh random password per node.

## Actuator Commands (Phase 2)

```
farm/{node_id}/commands/{command_type}
```

| Example topic | Description |
|---|---|
| `farm/zone-01/commands/irrigate` | Irrigation valve command for zone 01 |
| `farm/coop/commands/coop_door` | Coop door open/close command |

### Actuator Command Payload (JSON)

```json
{
  "command_id": "uuid-v4",
  "action": "open",
  "zone_id": "zone-01"
}
```

| Field | Type | Description |
|---|---|---|
| `command_id` | string (UUID v4) | Unique command identifier for ack correlation |
| `action` | string | `"open"` or `"close"` |
| `zone_id` | string | Zone or device identifier |

## Command Acknowledgments (Phase 2)

```
farm/{node_id}/ack/{command_id}
```

### Ack Payload (JSON)

```json
{
  "command_id": "uuid-v4",
  "node_id": "zone-01",
  "status": "confirmed",
  "ts": "2026-04-09T12:00:00Z"
}
```

| Field | Type | Description |
|---|---|---|
| `command_id` | string | Matches the command's `command_id` |
| `node_id` | string | Node that executed the command |
| `status` | string | `"confirmed"` or `"failed"` |
| `ts` | string (ISO 8601 UTC) | Timestamp of ack |

| Property | Value |
|---|---|
| QoS (commands) | 1 |
| QoS (acks) | 1 |
| Retain | `false` |
